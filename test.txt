3) Ujednolicenie bboxów (koordynatów) do jednego formatu

Najczęściej bbox występuje jako:

bbox: {x, y, width, height} albo

boundingBox: {left, top, right, bottom} albo

polygon: [{x,y}...]

Poniżej helpery w Pythonie do normalizacji (działają na obiekcie doc już z json.loads):

import re
from typing import Dict, Any, Optional, List, Tuple

def norm_bbox(obj: Dict[str, Any]) -> Optional[Dict[str, float]]:
    """
    Zwraca bbox jako {x0,y0,x1,y1} w tym samym układzie, w którym daje parser.
    (czy to jest 0..1 czy punkty/piksele – zależy od parsera; zachowujemy to 1:1)
    """
    if obj is None:
        return None

    # wariant: left/top/right/bottom
    bb = obj.get("boundingBox") or obj.get("bbox")
    if isinstance(bb, dict):
        if all(k in bb for k in ("left","top","right","bottom")):
            return {"x0": float(bb["left"]), "y0": float(bb["top"]), "x1": float(bb["right"]), "y1": float(bb["bottom"])}
        if all(k in bb for k in ("x","y","width","height")):
            x0, y0 = float(bb["x"]), float(bb["y"])
            return {"x0": x0, "y0": y0, "x1": x0 + float(bb["width"]), "y1": y0 + float(bb["height"])}

    # wariant: polygon (lista punktów)
    poly = obj.get("polygon") or obj.get("vertices")
    if isinstance(poly, list) and poly:
        xs = [p.get("x") for p in poly if "x" in p]
        ys = [p.get("y") for p in poly if "y" in p]
        if xs and ys:
            return {"x0": float(min(xs)), "y0": float(min(ys)), "x1": float(max(xs)), "y1": float(max(ys))}

    return None
4) Spis treści (TOC) + koordynaty
4.1 Skąd wziąć “linie” do TOC

Jeśli masz pages[].lines[] albo pages[].blocks[] z tekstem, bierzemy elementy tekstowe z bboxami i filtrujemy regexem “coś …… numer strony”.

Poniższy kod próbuje znaleźć listę elementów tekstowych na stronach (obsługuje kilka popularnych nazw pól):

def iter_text_items(doc: Dict[str, Any]):
    pages = doc.get("pages") or []
    for pi, page in enumerate(pages, start=1):
        # możliwe pola z liniami/blokami:
        candidates = []
        for k in ("lines", "blocks", "paragraphs", "spans"):
            v = page.get(k)
            if isinstance(v, list):
                candidates.extend(v)

        for it in candidates:
            text = it.get("text") or it.get("content") or it.get("value")
            if not text or not isinstance(text, str):
                continue
            bb = norm_bbox(it)
            yield {
                "page": pi,
                "text": text.strip(),
                "bbox": bb,
                "raw": it
            }

toc_line_re = re.compile(r"""
^
(?P<title>.*?)
(?:\.{2,}|\s{2,}|\t+)      # kropki lub większy odstęp
(?P<pageno>\d{1,4})
\s*$
""", re.VERBOSE)

# Heurystyka: często TOC jest na początku; tu możesz ograniczyć np. do pierwszych N stron
MAX_TOC_PAGES = 5

items = [x for x in iter_text_items(doc) if x["page"] <= MAX_TOC_PAGES]

toc = []
for x in items:
    m = toc_line_re.match(x["text"])
    if not m:
        continue
    title = m.group("title").strip(" .\t")
    pageno = int(m.group("pageno"))
    if len(title) < 3:
        continue
    toc.append({
        "toc_page": x["page"],            # strona, na której jest spis treści
        "entry_title": title,
        "target_page": pageno,            # strona docelowa z TOC
        "bbox": x["bbox"],                # bbox linii w TOC (podświetlenie w TOC)
        "text": x["text"]
    })

Wynik toc to lista wpisów z koordynatami linii w spisie treści.

5) Tabele + koordynaty komórek

Jeżeli parser zwraca tables[] lub pages[].tables[], zwykle znajdziesz:

bbox tabeli,

komórki (cells[]) z tekstem, wiersz/kolumna oraz bbox.

Kod poniżej:

zbiera tabele z dokumentu (kilka możliwych ścieżek),

zwraca listę komórek z (page, table_index, row, col, text, bbox).

def iter_tables(doc: Dict[str, Any]):
    # wariant A: top-level tables
    if isinstance(doc.get("tables"), list):
        for ti, t in enumerate(doc["tables"], start=1):
            yield (None, ti, t)

    # wariant B: tables per page
    pages = doc.get("pages") or []
    for pi, page in enumerate(pages, start=1):
        if isinstance(page.get("tables"), list):
            for ti, t in enumerate(page["tables"], start=1):
                yield (pi, ti, t)

tables_cells = []
for page_no, table_no, t in iter_tables(doc):
    t_bbox = norm_bbox(t)
    cells = t.get("cells") or t.get("tableCells") or []
    for c in cells:
        text = (c.get("text") or c.get("content") or "").strip()
        row = c.get("rowIndex") if c.get("rowIndex") is not None else c.get("row")
        col = c.get("columnIndex") if c.get("columnIndex") is not None else c.get("col")
        c_bbox = norm_bbox(c)
        # Jeśli komórka nie ma bbox, czasem jest w polu "layout"/"span"
        if c_bbox is None and isinstance(c.get("layout"), dict):
            c_bbox = norm_bbox(c["layout"])

        tables_cells.append({
            "page": page_no or t.get("pageNumber") or None,
            "table_index": table_no,
            "table_bbox": t_bbox,
            "row": int(row) if row is not None else None,
            "col": int(col) if col is not None else None,
            "cell_text": text,
            "cell_bbox": c_bbox,
        })

Jeśli len(tables_cells) == 0, to znaczy że Twoja wersja ai_parse_* nie materializuje tabel jako struktury. Wtedy opcja “co lepiej radzi sobie z tabelką” zwykle polega na:

wzięciu elementów tekstowych + bboxów

wykryciu “siatki” po pozycjach X/Y (klastrowanie po kolumnach/wierszach)

i zbudowaniu tabeli z layoutu (to da się zrobić deterministycznie, bez LLM).

Jeżeli chcesz, mogę dać gotowy algorytm klastrowania kolumn/wierszy po bboxach (działa dobrze na tabelach z wyraźnymi kolumnami).
